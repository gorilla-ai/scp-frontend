Object Types

KeyLines uses JavaScript objects for defining visual appearances.

Chart

The chart object lets you load JSON data into the chart. In this example, only the 'type', 'id', 'id1' and 'id2' properties are required:

var data = {
 type: 'LinkChart',          //this must be exactly this string
 items: [
   { 
     id: 'node1',             //'node1' is the identity of the node
     type: 'node',            //the type of the item: must be 'node' for nodes
     t: 'label',              //the label to be used under the icon (optional)
     e: '5'                   //the enlargement factor of the node (optional)
   },
        ...
   {
     id: 'link1',             //'link1' is the identity of this link
     id1: 'node1',            //the identity of the node at one end
     id2: 'node2',            //the identity of the node at the other end
     type: 'link',            //the type of the item: must be 'link' for links
     t: 'label',              //the label to be used for the link (optional)
     c: 'rgb(0, 0, 255)'      //the colour of the link (optional)
   }
 ]
};

Shapes

Shapes are a special case of nodes:

  • Shapes are drawn underneath ordinary nodes and links
  • Shapes are either images, boxes or circles
    • If the 'u' property is defined the node will be shown as an image

Nodes displayed as shapes use the same object properties as nodes, but with additional 'w' and 'h' properties to define the width and height of the resulting shape.

To display image shapes without stretching, set the width and height to the actual image size.

   {
     id: 'image1',            //the identity of the image
     t: 'World',              //the label to be used under the image
     type: 'node',            //images are also type 'node'
     u: 'im/map.png',         //the url of the image
     x: 100,                  //the x position (measured to the centre of the image)
     y: 150,                  //the y position (measured to the centre of the image)
     w: 500,                  //the width that the image should be stretched to
     h: 211                   //the height that the image should be stretched to
   }

These node properties do NOT apply to shapes:

  • ci: shapes cannot have a cutout circle from the original image
  • du: shapes cannot act as dummy nodes
  • e: shapes do not have an enlargement factor applied
  • ha: shapes do not have halos

Bubbles

Bubbles can be shown on nodes and links. You can only show one bubble per item.

Bubbles can be shown in one of four positions:

  • North East (ne)
  • South East (se)
  • South West (sw)
  • North West (nw)

Bubbles are added to items as follows:

var data = {
 type: 'LinkChart',
 items: [
   {
     id: 'node1', t: 'label', type: 'node', u: 'person.png', x: 100, y: 150,
     bu: {
       c: 'rgb(255, 0, 0)',         //the bubble fill colour
       p: 'ne',                     //bubble in NE position. use 'se', 'sw', 'nw' for the other positions
       t: 'Bubble'                  //the bubble text
     }
   }
 ]
};

Glyphs

Glyphs can be shown on nodes, links and bubbles.

Each glyph can display a coloured circle, an image or text. Refer to imageAlignment section in chart.options() for moving glyph icons from their default positions.

Glyphs on Nodes

Glyphs are drawn along a node's border. They are added as an array of objects to the 'g' property of the node.

There are two styles of positioning. Use one of the following:

  • compass points:
    • north (n)
    • north east (ne)
    • east (e)
    • south east (se)
    • south (s)
    • south west (sw)
    • west (w)
    • north west (nw)
  • integers in the range 0-359. They represent the angle from the centre of the node, measured in degrees clockwise from north.

Glyphs on a single node must use the same style of positioning (i.e., they must all use compass points or all use angles). In an array, if subsequent glyphs don't use the same positioning style as the first glyph, they won't be drawn.

You can have as many glyphs on a node as you like, but only one glyph is allowed in each position.

var data = {
 type: 'LinkChart',
 items: [
   {
     id: 'node1', t: 'label', type: 'node', u: 'person.png', x: 100, y: 150,
       g: [
         {                          
         c: 'rgb(255, 0, 0)',         //the glyph fill colour
         p: 'ne',                     //glyph in NE corner
         t: '!'                       //the glyph text
         }
       ]
   }
 ]
};

Glyphs on Links

Links show the glyphs next to the label. You can add as many glyphs as you like. Glyphs are added as an array of objects to the 'g' property:

var data = {
 type: 'LinkChart',
 items: [
   {
     id:'link1', id1: 'node1', id2: 'node2', type: 'link',
       g: [                             //an array of glyphs, shown left to right
         {
           c: 'rgb(255, 0, 0)',         //the glyph fill colour
           t: '!'                       //the glyph text
         }
       ]
   } 
 ]
};

Font Icons

Font Icons can be used as icon images on nodes and as glyphs for nodes and links.

In order to use a font icon, you must make sure it is included on your HTML page and that it is referenced by KeyLines by setting the option iconFontFamily.

Notes:

  • If you're using KeyLines 3.1 or later, you need a third party font loader to use font icons. For more information, see Font Loading.
  • Before it can use a font icon, KeyLines needs to examine the icon file. Your browser won't let KeyLines do this if the icon you're referencing is on a different domain to keylines.js. For details of how to fix this, see Cross-Origin Images.

Font Icons are coloured as follows: 

var data = {
 type: 'LinkChart',
 items: [
   {
     id: 'node1', t: 'label', type: 'node', x: 100, y: 150,
       fi: {                          
         c: 'rgb(255, 0, 0)',         //the icon fill colour
         t: '☺︎'                       //the icon text
        }
   }
 ]
};

The 't' property can accept either the text to render, or the UNICODE position of it:

var data = {
 type: 'LinkChart',
 items: [
   {
     id: 'node1', t: 'label', type: 'node', x: 100, y: 150,
       fi: {                          
         c: 'rgb(255, 0, 0)',                      //the icon fill colour
         t: KeyLines.getFontIcon('.icon-user')     //getFontIcon returns the UNICODE position
        }
   }
 ]
};

Font icons are displayed in a similar way to images, but unlike images it is possible to animate their colour.

If an image is also specified for a node, using the 'u' property, the image takes precedence and the font icon will not be displayed.

To set the position and scale of font icons, use the imageAlignment chart option.

Halos can be shown on nodes. A node can have up to ten halos, on properties named ha0 to ha9. Halos are not shown on shape nodes.

     ha0: {

       r: 100,                      //the halo radius
       w: 20                        //the halo width
     }

Donuts

Nodes can have donuts around their borders. They can be split into segments to represent relative quantities, similar to the way sectors work on a pie chart. Donuts can only be added to nodes with 'circle' shape.

Donuts are added to nodes as follows:

var data = {
 type: 'LinkChart',
 items: [
   {
     id: 'node1', type: 'node', c: '#aaa', x: 100, y: 150,
     donut: {
       // the values for each donut segment

Time Bar

The time bar object lets you load JSON data in the following format into the time bar:

var data = {
 items: [
   { id: 'id1',         //'id1' is the identity that the item relates to
     dt: [...],         //a timestamp or array of timestamps - either Date objects or numbers
     v:  [...],         //an optional value or array of values for each timestamp
                        //values must be numbers, there must be one for each timestamp
   },
   { id: 'id2',
     dt: [...],
     v:  [...]
   },
   ...
 ]

dt
required

required
string
The identity that this item is associated with. It must not end with a '\' character.

Each time bar item has a dt property that holds a time bar entry or an array of time bar entries. Each entry may be one of the following:

  • a Date object or millisecond number, representing an instant in time
  • a TimePeriod object, representing a period of time

layout(name, options, callback)layout(name, options)

chartasync
callbackpromise

The layout function positions the nodes of the chart.

The layouts are:

  • Standard: an efficient force-directed graph layout.
  • Hierarchy: lays out tree-like data top down, usually from a specified node.
  • Beta Sequential: places nodes at distinct levels and minimises crossed links.
  • Lens: places the node in a circle-like grid, with connected nodes next to each other.
  • Radial: places nodes in concentric circles.
  • Structural: places nodes which are structurally similar together in the network.
  • Tweak: adjusts node positions in a force-directed way.

Options can be passed in the second argument. For example:

chart.layout('standard', {fit: true, animate: true, tidy: true});

In map mode, the layout is not performed.

Hidden items do not change their positions. When re-showing hidden items consider running a layout to ensure the chart looks good afterwards.

The layout call is asynchronous - the function will return before the layout has completed. To discover the progress of the layout use the progress event. Use a callback function to detect when the layout has finished.

The arrange function is an alternative way to position nodes. It places nodes in close proximity into a specified shape.

Events allow you to customise your application by responding to user actions on the chart surface. To bind to events, use chart.bind.

Target Sub-Widgets

You can bind events to certain objects on the chart, as well as to specific parts of nodes, links, halos, glyphs, donuts and shapes. These parts are called sub-widgets, and include the individual arrowheads of a link, the segments of a donut, or a node’s text label.

To bind to events, pass the sub-widget ID through any event with a 'sub' parameter. For example, to bind a click event to the first segment of a donut, use:

chart.bind('click', function (id, x, y, button, sub) {
  if(sub === 'do0') {

  } 
});

maxRange
string or MaxRange
default: 'auto'

Specifies the time range at the time bar's maximum zoom level, for example with a maxRange fixed to { units: 'hour', value: 6 } the time bar will not zoom out beyond six-hour intervals. This is useful if your dataset spans a large time range and you want to limit the amount of data visible.

For the 'auto' default, KeyLines calculates the range between the data's earliest and latest dates/times and multiplies it by two.
